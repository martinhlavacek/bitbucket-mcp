# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

1. **Do NOT** open a public issue
2. Email the maintainer directly or use GitHub's private vulnerability reporting
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

We will acknowledge receipt within 48 hours and provide a detailed response within 7 days.

## Security Best Practices for Users

When using Bitbucket MCP:

1. **Never commit credentials** - Use environment variables for Bitbucket App Passwords
2. **Use App Passwords** - Don't use your main Bitbucket password
3. **Limit permissions** - Create App Passwords with minimal required scopes
4. **Rotate credentials** - Regularly rotate your App Passwords
